\section{数字集成电路后端设计全流程}

数字集成电路后端设计（Physical Design）是连接前端逻辑设计与芯片制造的核心环节，其核心目标是将前端综合生成的门级网表，转换为满足时序、功耗、面积与可制造性约束的物理版图，最终输出可用于流片的图形数据库文件（Graphic Database System II, GDSII）。随着集成电路工艺进入7\,nm及以下先进节点，芯片规模达到百亿级晶体管级别，后端设计复杂度显著提升。全局布线（Global Routing）作为后端设计中影响时序收敛、绕通率与良率的关键步骤，其技术优化已成为后端设计的重要研究方向。

完整的数字集成电路后端设计全流程可分为7个核心阶段。各阶段前后衔接、迭代优化，共同保证芯片设计的最终交付。图~\ref{fig:chap2-flow-overview-placeholder}给出了流程框图。

\section{全局布线技术基础}

全局布线是详细布线前的关键预布线环节。其核心是在不确定具体几何形态的前提下，为全部线网生成可行全局路径并预分配资源，在兼顾线长、时序与拥塞的同时，为详细布线提供高质量指导，降低后续迭代成本。

\subsection{全局布线的核心定义与网格约束}

\subsubsection{全局布线的网格模型}

工业界主流全局布线工具普遍采用格点网格模型：将核心区划分为等大小矩形网格单元。每个格点边界对应布线通道，通道具有固定容量（Capacity），由金属层可用资源、线宽与线距等工艺参数共同决定。当实际需求超过容量时即产生溢出（Overflow）。

\subsubsection{全局布线的核心目标}

全局布线是多目标优化问题，主要目标包括：
\begin{enumerate}
    \item \textbf{绕通率优先}：保证线网可行路径并控制整体拥塞；
    \item \textbf{资源代价最小化}：在满足约束前提下降低总线长、通孔数量等指标；
    \item \textbf{时序优化}：优先保障关键路径线网延迟；
    % \item \textbf{资源均衡}：均衡不同区域与不同层资源占用；
    % \item \textbf{可制造性优化}：控制通孔数量并合理使用高层长距离走线。
\end{enumerate}

\subsubsection{金属层布线方向与跨层约束}

在全局布线中，不同金属层通常具有不同的布线方向：一组金属层以横向为布线方向，另一组金属层以纵向为布线方向，并在层间交替配置。该约束为全局布线环节的基础规则约束，可提升资源利用率，并为跨层绕行提供结构化通道。

当线网从一个布线方向层切换到另一布线方向层时，需要通过通孔（Via）实现层间连通。通孔数量与跨层切换次数会直接影响延迟、拥塞和制造代价，因此在全局布线阶段需要同时考虑同层路径选择与跨层切换开销。图~\ref{fig:chap2-cross-layer-constraint}给出了格点跨层布线约束示意。

\subsection{全局布线流程总览}

在工程实现中，全局布线通常可划分为四个阶段：
\begin{enumerate}
    \item \textbf{树生成阶段（Tree Generation）}：构建多端线网的初始连通树拓扑；
    \item \textbf{模板布线阶段（Pattern Routing）}：基于预定义模板快速生成候选路径并完成初始层分配；
    \item \textbf{撤销与重布阶段（Rip-up and Reroute）}：针对拥塞线网执行局部拆除与增量重布，缓解热点区域溢出；
    \item \textbf{迷宫布线阶段（Maze Routing）}：对仍未完成或难布通线网进行搜索式补充布线，提升最终绕通率。
\end{enumerate}

上述四个阶段考虑粒度逐渐变细，但是由于考虑粒度越细的算法速度越慢，因此在实际流程中，需要先采用粗粒度的算法，根据前一阶段粗粒度结果所造成的拥塞情况，采用细粒度的布线算法逐步降低全局拥塞。

\subsection{树生成阶段（Tree Generation）}

树生成阶段是全局布线的起点，其任务是将多端线网转换为连通树拓扑，确定引脚连接关系与斯坦纳点（Steiner Node）位置，为后续模板布线与拥塞优化提供拓扑基础。在曼哈顿平面场景下，多端线网线长最优解通常对应直角斯坦纳最小树（Rectilinear Steiner Minimum Tree, RSMT）。



Flute（Fast Lookup Table Based Rectilinear Steiner Minimal Tree）算法是工业界与学术界应用广泛的RSMT构建算法，在主流EDA流程中有良好工程实践。其核心优势是对小引脚数线网可高效获取高质量拓扑，并对大引脚数线网通过分治策略保持较优质量与速度平衡。

Flute流程包括：
\begin{enumerate}
    \item \textbf{预处理阶段}：针对小规模引脚组合建立查找表（Look-Up Table, LUT）；
    \item \textbf{在线处理阶段}：按引脚规模选择查表或分治合并策略，输出近似最优RSMT。
\end{enumerate}

图~\ref{fig:chap2-flute-flow}展示了Flute算法的核心处理链路：以线网引脚集合作为输入，通过查找表快速匹配拓扑，输出近似最优RSMT结果。

\subsection{模板布线与层分配（Layer Assignment）}

模板布线通过预定义布线模板将树边快速映射到G-Cell网格，是全局布线效率的核心来源。其特点是无需全图搜索即可覆盖大部分线网，再将少量困难线网交由Maze Routing处理。

\subsubsection{常用模板布线走线模板}

常见模式包括：
\begin{enumerate}
    \item \textbf{L型模板}：单拐点、实现简单，适合低拥塞短连线；
    \item \textbf{Z型模板}：双拐点模板，保持曼哈顿最短线长且具备更强绕障能力；
    \item \textbf{U型模板}：通过额外转折实现回绕，适合局部拥塞或障碍场景下的路径调整。
\end{enumerate}

\subsubsection{层分配（Layer Assignment）}

在层分配阶段，本文聚焦基于动态规划（Dynamic Programming, DP）的重分配算法。该算法将线网从起点到终点的候选层间转移关系建模为有向无环图（Directed Acyclic Graph, DAG），并在该图上求解最小代价路径。图~\ref{fig:chap2-dp-layer-reassignment}给出了动态规划层分配示意。

\begin{figure}[!htbp]
    \centering
    \includegraphics[width=0.80\textwidth]{image/chap02/动态规划层分配.pdf}
    \caption{基于动态规划的层分配示意图}
    \label{fig:chap2-dp-layer-reassignment}
\end{figure}

其主要流程如下：
\begin{enumerate}
    \item 根据可布线通道构造有向无环图；
    \item 在有向无环图基础上，根据边与边之间的走线权重，利用动态规划算法获得布线代价最小的路径。
\end{enumerate}



动态规划方法可用于单线网层分配最优化。设路径分段为$N$，金属层数为$M$，定义状态$dp[i][k]$表示第$i$段分配到第$k$层时的最小代价，则状态转移为：
\begin{equation}
    dp[i][k] = \min_{1 \leq j \leq M}\left( dp[i-1][j] + cost_{switch}(j,k) + cost_{layer}(i,k) \right)
\end{equation}
其中，$cost_{switch}$表示层切换代价，$cost_{layer}$表示当前层延迟与拥塞综合代价。为与工程实现一致，下述伪代码将状态写作$\mathrm{mvc}(v,k)$，其含义等价于在顶点-层空间上的最小累计代价更新。

\begin{algorithm}[!htbp]
    \KwIn{DAG顶点$v$，子流集合$\{S_i\}_{i=1}^{m}$，候选边集$\{E_i\}_{i=1}^{m}$，总层数$L$，固定层区间$[k_{\min}^{fix}, k_{\max}^{fix}]$}
    \KwOut{$\mathrm{mvc}(v,k)$与$\mathrm{bestPath}(v,k)$}
    \ForEach{$u\in P(v)$}{
        \textsc{UpdateMVC}$(u)$\;
    }
    计算过孔前缀代价$viaPrefix[k]$，其中$viaPrefix[k]=\sum_{t=0}^{k-1}via\_cost(v,t)$\;
    \For{$k_{\min}\leftarrow 0$ \KwTo $k_{\min}^{fix}$}{
        \For{$i\leftarrow 1$ \KwTo $m$}{
            $min\_stream\_cost(S_i)\leftarrow +\infty$\;
            $best\_stream(S_i)\leftarrow (-1,-1)$\;
        }
        \For{$k_{\max}\leftarrow k_{\min}$ \KwTo $L-1$}{
            \For{$i\leftarrow 1$ \KwTo $m$}{
                \ForEach{$u\in E_i$}{
                    $stream\_cost\leftarrow \mathrm{mvc}(u,k_{\max}) + edge\_cost(u\rightarrow v,k_{\max})$\;
                    \If{$stream\_cost < min\_stream\_cost(S_i)$}{
                        $min\_stream\_cost(S_i)\leftarrow stream\_cost$\;
                        $best\_stream(S_i)\leftarrow (u,k_{\max})$\;
                    }
                }
            }
            \If{$k_{\max} \geq k_{\max}^{fix}$}{
                $sum\_via\_cost\leftarrow viaPrefix[k_{\max}] - viaPrefix[k_{\min}]$\;
                $cost\leftarrow sum\_via\_cost + \sum_{i=1}^{m} min\_stream\_cost(S_i)$\;
                \If{$cost < \mathrm{mvc}(v,k_{\max})$}{
                    $\mathrm{mvc}(v,k_{\max})\leftarrow cost$\;
                    $\mathrm{bestPath}(v,k_{\max})\leftarrow \{best\_stream(S_i)\}_{i=1}^{m}$\;
                }
            }
        }
        \For{$k\leftarrow L-2$ \KwTo $k_{\min}$（降序）}{
            \If{$\mathrm{mvc}(v,k+1) < \mathrm{mvc}(v,k)$}{
                $\mathrm{mvc}(v,k)\leftarrow \mathrm{mvc}(v,k+1)$\;
                $\mathrm{bestPath}(v,k)\leftarrow \mathrm{bestPath}(v,k+1)$\;
            }
        }
    }
    \caption{基于动态规划的层重分配最小代价更新伪代码}
    \label{alg:chap2-dp-layer-assignment}
\end{algorithm}

\subsection{撤销与重布阶段（Rip-up and Reroute）}

模板布线与层分配完成后，部分线网仍可能集中穿越高拥塞边，导致局部溢出持续存在。撤销与重布阶段通过识别拥塞线网，局部拆除，生成绕线路径的流程，对高风险区域进行定向修复。

在该阶段中，边移动（Edge Shifting）用于在局部范围内调整树边穿越位置，尽量避开当前拥塞通道；若局部调整后仍无法有效降低溢出，则会留到迷宫布线阶段进行最细粒度的布线。撤销-重布阶段的本质是在原有的树拓扑基础上进行小幅度的细粒度修改以避开原有的拥塞区域集，以较小的代价逐步改善全局资源分布。图~\ref{fig:chap2-rrr-edge-shift}给出了边移动局部调整示意。

\subsection{迷宫布线阶段}

迷宫布线用于处理模板布线未完成线网，通过网格搜索提升最终绕通率。其优势是可行性强，以下是对迷宫布线算法及其变种算法的介绍：

\subsubsection{经典迷宫布线算法}

水波法迷宫布线算法本质是广度优先搜索（Breadth-First Search, BFS），在可行时保证最短路径。其步骤包括网格初始化、波前扩展与路径回溯。尽管鲁棒性强，但在大规模网格上计算与存储开销较高。如图~\ref{fig:chap2-lee-wave-placeholder}所示，水波法通过从起点向外扩散波前，逐层标记距离，直到抵达终点后回溯得到路径。

\end{figure}

\subsubsection{分阶段迷宫布线：粗粒度与细粒度折中方案}
由于前述经典迷宫布线算法直接全局搜索代价较高，因此工程中通常采用分层或局部化策略，由此产生了分阶段的迷宫布线方案。
分阶段迷宫布线将求解拆分为粗粒度全局搜索与细粒度局部搜索两阶段：
\begin{enumerate}
    \item \textbf{粗粒度阶段}：通过超级网格（Super Cell）降低状态规模并快速规划全局通路；
    \item \textbf{细粒度阶段}：在局部窗口内完成精细路径搜索，失败时扩大窗口或回退重规划。
\end{enumerate}

该方案可显著降低搜索规模，在效率与质量之间取得较好平衡。如图~\ref{fig:chap2-hier-maze-placeholder}所示，分阶段迷宫布线首先在粗粒度超级网格上规划大致路径，然后在细粒度网格内再次进行局部搜索以完成最终布线，避免了直接在全局细粒度网格上进行搜索的高昂代价。

\subsubsection{A*算法与启发式搜索优化}

A*算法是在Lee算法基础上引入启发式函数的有指导搜索方法，能够在保证最优性的前提下显著减少搜索节点数量，是当前迷宫布线中的重要技术，可视为经典迷宫布线在细粒度路径搜索层面的重要升级。

其核心原理如下：

A*的评价函数定义为：
\begin{equation}
    f(n) = g(n) + h(n)
\end{equation}
其中，$g(n)$为起点到当前节点的实际代价，$h(n)$为到终点的启发式估计。为保证全局最优，$h(n)$应满足可采纳性。对曼哈顿布线，常用启发式函数为：
\begin{equation}
    h(n) = |x_n - x_{end}| + |y_n - y_{end}|
\end{equation}

其实现流程包括：
\begin{enumerate}
    \item 初始化开放集合（Open）与关闭集合（Closed）；
    \item 每轮取$f(n)$最小节点扩展；
    \item 更新邻接节点代价与父指针；
    \item 抵达终点后回溯得到路径。
\end{enumerate}




\section{模块级布局技术基础}

模块级布局是在布局阶段以模块而非单个标准单元（Standard Cell）作为主要优化对象的技术路线。其基本思想是先确定模块间的全局连接关系，再在模块位置以及模块边界层面进行修正，最后将模块布局约束引入到布局流程中。该策略能够在大规模设计中显著压缩搜索空间，在计算负载增加有限的情况下提升布局结果的质量。

\subsection{模块级布局问题定义}

给定电路网表、芯片边界、输入输出端口位置以及工艺规则，模块级布局需要确定模块中心位置、边界形态与模块间相对拓扑，使结果在满足合法性约束的前提下兼顾性能-功耗-面积（Performance, Power, Area, PPA）与布线质量。其本质是典型的多目标、多约束组合优化问题，常见目标包括：
\begin{enumerate}
    \item 最小化跨模块互连线长；
    \item 确保模块形状的张力适中，避免过度拉伸或压缩；
    \item 保持模块不发生几何非法重叠或出界。
\end{enumerate}

然而，模块级布局规划算法的速度提升是以牺牲部分细节优化能力为代价的，并且即使是尽可能多的考虑布局约束，也无法完全覆盖后端工具的约束要求。因此，模块级布局规划结果与后端工具输入之间存在各种合法化问题导致的错配，这也是本文针对粗粒度的布局仍然存在的不足的研究切入点。

\subsection{网表分割与模块建模}

模块级布局的第一步是将原始电路网表（Netlist）转换为可分割的数据结构。具体而言，将每个标准单元（Standard Cell）或宏单元（Macro）抽象为节点（Node），将节点间的连接关系抽象为线网（Net）或超边（Hyperedge），从而得到用于分割求解的模块连接图。

在该图结构上，可利用图划分工具（如hMetis）按给定规模约束自动完成模块划分。划分目标通常是在控制模块面积平衡的同时，尽量提高模块内连接密度并降低模块间割边数量。经过分割后，原始高复杂度网表被降阶为模块级连接关系，后续布局优化的搜索空间显著收敛。

从工程输出看，该步骤通常会生成两类关键结果：
\begin{enumerate}
    \item 模块归属关系：每个Cell到模块的映射；
    \item 模块间连接关系：模块级线网权重与连接拓扑；
    % \item 模块规模信息：各模块面积与边界范围估计。
\end{enumerate}

\subsection{模块级布局优化}

在分割得到的模块基础上，第二步采用解析式粗粒度布局完成模块级优化。该阶段不以单个标准单元为直接优化对象，而以模块中心位置、模块边界参数与模块接口位置等为主要变量，在全局层面联合优化线长、重叠、边界与分布均衡等目标。

模块级布局优化可采用分阶段权重调度策略：前期优先确定模块相对拓扑并压缩主干线长；中期逐步提高合法性约束权重以消除重叠并满足边界条件；后期强化版图填充与分布均衡，使模块在版图内形成更稳定的几何排布。

需要指出的是，解析式粗粒度结果并不能直接等价为后端工具输入，而是需要以布局放置约束的形式对布局的结果进行引导。

图~\ref{fig:chap2-modular-placement-optimization}展示了模块级布局优化的过程，首先在优化之初，模块的分布都是初始化的，它们可能会边界不清晰、发生重叠，或者放置区域位置不对。
在经过解析式模块级布局算法优化之后，这些模块的区域将会逐渐填满整个版图，并且不同模块的放置位置也会更加合理。除此以外，模块级布局优化算法建模并考虑了众多约束项，经过优化最终会获得综合代价相对较小的模块级布局结果。
