# 第四章 基于模拟退火的模块化布局细粒度设计空间探索



## 4.1 研究背景

在超大规模集成电路（Very Large Scale Integration, VLSI）后端流程中，布局（Placement）与全局布线（Global Routing）是强耦合的两个阶段。前者决定单元空间分布，后者在给定空间资源上完成连线搜索。若布局阶段对拥塞热点、跨层连线密度和关键路径拓扑考虑不足，即使后续布线算法足够先进，也往往只能在受限设计空间中被动修复，导致线长（Wirelength）、拥塞（Congestion）和运行时间（Runtime）同时恶化。

第三章已经围绕动态拥塞感知初始布线给出了质量与效率协同优化方案。本章工作进一步向上游延伸，研究如何在已有模块化布局结果的基础上，通过外层设计空间探索（Design Space Exploration, DSE）获得对后续布线更友好的布局约束输入。该思路通过低侵入方式对布局结果进行增量优化，与各项布局优化算法耦合度低，可以较容易地插入到任意模块化布局结果中，使后续布线在更优初态上运行。



## 4.2 细粒度布局优化的必要性与挑战

### 4.2.1 布局问题中的约束复杂性

集成电路布局本质上是多约束、多目标、强耦合优化问题。其典型优化目标为性能-功耗-面积（Performance, Power, Area, PPA）与可布线性（Routability）的综合平衡，同时必须满足工艺可制造性和设计规则。该问题已被广泛认为具有非确定性多项式困难（Non-deterministic Polynomial-time hard, NP-hard）属性。

从工程角度，布局约束可粗略分为三类（其实也不止三类，并且约束类型多样，不同的设计流程、制程节点都会导致约束的不同）：

1. 物理合法性层：边界约束、行对齐约束、非重叠约束、电源域约束。
2. 电气性能层：时序约束、功耗与电源完整性约束、信号完整性约束。
3. 流程耦合层：与时钟树综合、全局布线、详细布线和签核指标的跨阶段一致性约束。

布局问题具有极高的复杂性，设计搜索空间极大，难以精确的建模，也难以在高质量的同时保持高效率。基于效果与速度的权衡，布局优化方法大致可以分为两类，一类是基于解析式优化的粗粒度模块化方法，另一类是基于模拟退火的细粒度全局搜索方法。前者效率高但调整粒度较粗，后者调整粒度细但效率较低。两者之间存在明显的“质量-效率”权衡。


### 4.2.2 解析式模块化布局方法概述

解析式布局（Analytical Placement）通过构建可微目标并借助高效优化器，在大规模设计中具有显著效率优势。代表性路线包括 EPlace、RePlAce、DREAMPlace 等布局框架。其核心特点是：

1. 以连续变量近似离散布局问题，提高求解效率。
2. 通过密度势场等机制处理重叠与均匀性。
3. 通过 GPU 并行加速实现高吞吐优化。

在模块化布局场景中，解析式方法进一步通过模块边界与模块间关系建模，形成宏观约束输入。该类结果对于缩小可搜索空间、改善全局拓扑具有明显价值，也是本章外层优化的输入基础。

### 4.2.3 粗粒度模块化优化的局限性

尽管粗粒度模块化方法速度快、可扩展性强，但在工业流程落地时仍存在三个痛点：

1. 指标建模近似：为保持可微与并行，部分目标需要近似表达，难覆盖工具内部真实代价。
2. 几何分辨率不足：宏观边界对局部拥塞通道与关键跨区连线的表达能力有限。
3. 工具约束失配：商业工具接受的约束类型与几何形态有限，原始边界需再次合法化，可能引入额外自由度。

因此，仅依赖粗粒度的解析式布局算法难以稳定获得后端收益；仅依赖纯模拟退火（Simulated Annealing, SA）全局搜索又会带来极高的计算成本。二者之间需要中间层：在已有较优粗粒度解附近，用可控代价进行细粒度增量搜索。

### 4.2.4 细粒度模块化优化问题定义

设已有模块化布局输出为初始约束集合 $C_0$，布局工具在该约束下执行布局布线后得到质量指标向量：

$$
\mathbf{Q}(C)=\left[W(C),V(C),T(C),O(C),R(C)\right]
$$

其中 $W$ 为总线长，$V$ 为通孔数，$T$ 为总运行时间，$O$ 为拥塞相关指标（如 overflow），$R$ 为可布线性相关评价。本文关注在给定预算内寻找改进约束：

$$
C^*=\arg\min_{C\in \mathcal{N}(C_0)} F(\mathbf{Q}(C))
$$

其中 $\mathcal{N}(C_0)$ 表示围绕 $C_0$ 的可行邻域，$F$ 为多指标加权目标。该问题不追求全局最优，而追求在可接受开销内获得稳定增益。

### 4.2.5 纯粗粒度或纯模拟退火路线的不足

在工程实践中，常见两种极端路线：

1. 仅使用粗粒度解析式结果，不再进行后处理搜索。
2. 从较弱初值出发，直接使用大规模模拟退火进行全空间搜索。

前者的问题是“可调度性不足”。粗粒度结果虽然整体拓扑合理，但当设计进入工具内部的真实代价面时，缺少足够自由度做针对性修复。后者的问题是“计算预算不可控”。若在全空间进行 SA，候选数与评估次数迅速增长，而每次评估都依赖重型后端流程，整体代价很难接受。

本章采用的“粗粒度先验 + 细粒度设计空间探索”本质是二者折中：

1. 用粗粒度结果提供较好初值与搜索边界。
2. 用细粒度邻域修复工具可见但上游难显式建模的问题。
3. 用预算约束下的迭代评估确保收益-成本比可控。

通过基于模拟退火的细粒度设计空间探索方法，补齐解析式的粗粒度优化与纯模拟退火的全局搜索之间的空白，实现质量与效率的协同提升。

## 4.3 基于模拟退火的细粒度设计空间探索方法

### 4.3.1 总体流程

本章方法输入为已有模块化边界及其对应网表信息，输出为可被主流商业后端布局布线工具接受的优化后约束文件。流程分为六步：

1. 读取初始约束并解析分组约束的几何与类型信息。
2. 对非规则边界执行合法化映射，得到可编辑约束表示。
3. 设计满足工具规则的邻域操作集合。
4. 通过单轮模拟退火在局部邻域内搜索更优约束。
5. 通过多轮重启策略将每轮最优解作为下一轮初始解。
6. 调用后端布局布线工具执行真实流程评估，记录日志并输出最优约束。

该流程强调“搜索-评估”闭环：每个候选约束都由真实后端工具评估得到真实指标，具有良好的工程化意义。

### 4.3.2 基于分组约束的引导式布局机制

主流后端布局布线工具通常支持通过分组约束引导实例分布。每个分组约束由两部分定义：

1. 几何区域：由多边形边界指定。
2. 约束类型：可按约束强度分为软约束、中性约束、硬约束。

三类约束可理解为不同“约束等级”：

1. 软约束：属于该分组的实例可以超出该分组边界，不属于该分组的实例也可以进入该分组边界。
2. 中性约束：属于该分组的实例应保持在该分组边界内，不属于该分组的实例可以进入该分组边界。
3. 硬约束：最强等级的约束类型，属于该分组的实例必须保持在该分组边界内，不属于该分组的实例不允许进入该分组边界。

上述三种约束强度是商业后端设计工具长期支持的经典表达方式，具有广泛工程适用性。基于分组约束的引导式布局机制为本章方法提供了天然接口：将模块化布局结果映射为一组分组约束，在此基础上进行细粒度调整，最终输出符合工具要求的约束文件供后续流程使用。
在本章中，模块化结果被映射为一组分组约束后，细粒度布局优化算法并不改动实例级网表，仅修改分组约束的几何边界与约束类型，从而实现低耦合度的增量优化。这种策略在工程上更稳定，也更易和既有流程集成。

### 4.3.3 自由边界到工具可用约束的合法化

#### 4.3.3.1 合法化必要性

模块化方法输出的边界可能是自由多边形，而主流后端工具对约束多边形通常有明显曼哈顿化倾向，即边需与坐标轴对齐。其约束根源在于，在集成电路后端设计中，布局规划阶段通常已经确定版图行（row）的高度（或宽度）。由于行结构存在，所有实例都需要规则地摆放在行内。直接输入非规则边界，容易导致约束解释歧义或非法。因此需要边界合法化。

#### 4.3.3.2 合法化策略

本章采用“条带切片 + 矩形重建”的思路。给定多边形 $P$，假设 row 的方向为横向排布，沿 $y$ 方向以固定步长 $h$ 进行切片，计算上下切片线与多边形边界交点，提取每个高度区间内有效横向范围，再重建为矩形序列。

在当前原型实现中，步长可与布局行（row）尺度对齐（以 ASAP 设计库为例，此约束值为 0.27um）。为保证约束稳健性，工程脚本中进一步提供包围盒化简路径，使边界在复杂情况下仍能稳定落地。

### 4.3.4 邻域操作设计

合法化后，候选约束的搜索通过邻域操作完成。结合本文方法，本章采用五类操作：

1. 约束类型改变：随机调整分组约束类型（软约束/中性约束/硬约束）。
2. 边微调：随机选取局部边并向内或向外平移。
3. 边界扩展：在边界上增加矩形片段。
4. 边界收缩：移除边界上的矩形片段。
5. 整体平移：整体平移多边形。

上述操作覆盖“类型-形状-位置”三个维度，既能进行局部微调，也能进行结构性扰动。相较单一移动操作，组合邻域可显著提高跳出局部最优的概率。

设当前解为 $C_t$，邻域生成可写作：

$$
C_{t+1}=\Phi(C_t;\omega_t,\delta_t)
$$

其中 $\omega_t$ 为操作类型，$\delta_t$ 为操作强度（如平移距离、修改分组数量）。

### 4.3.5 目标函数与接受准则

#### 4.3.5.1 目标函数

依据工程实现，目标函数采用线长与几何规整性的加权形式：

$$
J(C)=\alpha \cdot \frac{W(C)}{W_0}+\beta \cdot (1-S(C))
$$

其中：

1. $W(C)$ 为该约束下评估得到的总线长。
2. $W_0$ 为初始线长，用于归一化。
3. $S(C)$ 为边界方正规整度评分。
4. $\alpha,\beta$ 为权重（示例配置为 0.7 与 0.3）。

该定义体现了本章方法的工程取向：既关注线长收益，也约束边界形态，避免极端不规则解对工具稳定性造成风险。

并且，由于设计的各项指标具体值均由实际布局后实际评估得到，能够非常真实的反映该布局所真实造成的各项代价，因此该目标函数具有很好的工程意义。并且该目标函数的设计也非常灵活，可以根据实际需求调整权重，甚至引入更多指标（如时序、拥塞等），为不同指标设置不同的权重，可以非常好的适应使用者的设计需求偏好。
#### 4.3.5.2 Metropolis 接受准则

若候选解使目标改善（$\Delta J\le 0$），则直接接受；否则按概率接受：

$$
P_{acc}=\exp\left(-\frac{\Delta J}{T}\right)
$$

其中 $T$ 为当前温度。该机制使算法在高温阶段具有探索性，在低温阶段具有收敛性。

### 4.3.6 单轮模拟退火中的动态强度调度

为了平衡“探索广度”与“收敛稳定性”，本章方法根据温度动态调整扰动强度：

1. 高温阶段：修改更多分组约束，增强全局探索。
2. 中温阶段：线性降低修改范围，逐步聚焦。
3. 低温阶段：仅修改少量分组约束，执行细粒度收敛。

在工程脚本中，这一思想具体落到“每轮修改分组数、每个分组修改次数、平移幅度”等参数上。即使某些参数在当前版本中固定，调度框架本身已经建立，为后续扩展（如自适应温控）预留了接口。

### 4.3.7 多阶段退火与多轮重启策略

单轮模拟退火常见问题是对初值敏感。为此，本章采用“内层迭代 + 外层轮次”的两层结构：

1. 内层 iteration：标准模拟退火搜索过程。
2. 外层 round：每轮结束后，将最优解复制为下一轮初值。

该策略相当于“带记忆的多次重启”，可在不过度放大单轮开销的前提下提升全局搜索能力。

对应伪代码如下：

```text
Algorithm 4-1 多轮细粒度约束优化流程
Input: 初始约束 C0, 轮数 R, 每轮迭代 K
Output: 全局最优约束 Cbest

Cbest <- C0
for r in [1..R]:
    Ccur <- Cbest
    初始化温度 T <- T0
    for k in [1..K]:
        生成邻域 Cnew <- Neighbor(Ccur, T)
        调用后端工具评估 J(Cnew)
        按 Metropolis 准则决定是否接受
        若 J(Cnew) 更优则更新本轮最优
        T <- cooling(T)
    end for
    用本轮最优更新 Cbest
end for
return Cbest
```

### 4.3.8 约束修复与可行性守护机制

SA 邻域扰动可能产生临时不可行解。为避免搜索被非法候选污染，本章在工程实现中引入可行性守护策略：

1. 几何合法性守护：邻域变换后立即检查边界是否满足基本几何条件（如宽度非负、顶点序列无异常）。
2. 多边形清理守护：对连续共线点执行去冗余处理，减少工具解析歧义。
3. 退化回滚守护：若某次变换导致明显失效，可直接回退到上一步合法边界。

### 4.3.9 遗传算法在模块化布局优化场景中的应用方式

为评估群体智能路线在本问题中的可行性，本文构建了遗传算法原型。该方法将分组约束配置视为可进化个体，个体中包含分组边界几何与约束强度信息；通过完整后端流程评估其质量，以线长等指标作为适应度反馈。

在模块化布局约束优化中，遗传算法可按以下步骤应用：

1. 个体编码与初始化：将初始模块化约束映射为若干候选个体，并通过随机扰动扩充初始种群，形成多起点搜索。
2. 真实指标评估：对每个个体执行完整评估流程，提取线长、通孔与运行时间等指标，计算适应度。
3. 父代选择与精英保留：通过竞争选择保留高质量个体，并将最优个体直接继承到下一代，避免优质解丢失。
4. 约束重组：从两个父代中重组部分分组约束，形成新候选，以扩大搜索多样性。
5. 变异扰动：对候选解执行约束类型改变、边微调、边界扩展、边界收缩与整体平移等操作，在局部空间中引入新结构。
6. 分阶段强度调度：随着代数增加，逐步降低变异范围与扰动幅度，实现由全局探索向局部收敛过渡。

上述流程说明，遗传算法能够在早期快速覆盖较大搜索空间，具备较强的全局探索潜力；但其代价是每一代都需要评估多个候选解，计算开销与评估预算高度相关。

### 4.3.10 模拟退火与遗传算法的差异及优劣分析

基于上述实现，可将模拟退火与遗传算法的差异归纳为以下四点：

1. 搜索组织方式不同：模拟退火沿单解轨迹逐步演化，遗传算法并行维护群体并通过群体多样性扩展搜索范围。
2. 新解产生机制不同：模拟退火依赖邻域扰动与概率接受；遗传算法依赖选择、重组与变异。
3. 开销结构不同：若种群规模为 $P$、迭代代数为 $G$、每代精英数为 $E$，遗传算法评估次数近似为

$$
N_{eval}^{GA}\approx N_0 + G\cdot(P-E)
$$

其中 $N_0$ 为初始种群评估开销；而模拟退火评估次数通常可写为

$$
N_{eval}^{SA}\approx R\cdot K
$$

其中 $R$ 为外层轮次、$K$ 为每轮迭代数。对于评估代价昂贵的问题，模拟退火更便于按预算精细控制。
4. 收敛行为不同：遗传算法更强调全局多样性，模拟退火更强调局部连续改进与稳定收敛。

从方法优劣看，遗传算法的优势在于全局探索能力强、对初值依赖相对较弱；不足在于参数耦合度高、评估开销大、收敛速度对种群配置敏感。模拟退火的优势在于预算可控、实现简洁、在较优初值附近易获得稳定增量收益；不足在于单轮搜索可能陷入局部最优。本文通过多轮重启与动态强度调度，部分缓解了模拟退火的局部最优风险。

结合本文已有测试结果，遗传算法在该场景下未表现出显著更强的优化收益，且整体运行速度明显慢于模拟退火。其核心瓶颈在于群体方法需要反复评估大量候选解，而每次评估都依赖高代价后端流程。因此，在“评估昂贵、强调单位时间收益、追求稳定可复现改进”的研究目标下，选择模拟退火作为本章主方法具有必要性与合理性。

## 4.4 工程实现与复杂度分析

### 4.4.1 工程脚本架构

本章方法以两个核心脚本实现：

1. 随机邻域修改.py：负责约束解析、合法化与邻域变换。
2. 评估驱动脚本：负责模拟退火主循环、调用后端评估工具、提取指标、记录日志。

脚本层次清晰，便于替换其中任一环节：例如可替换目标函数、加入新邻域、或切换评估工具。

### 4.4.2 关键实现细节

1. 约束修改以分组约束定义行为单位，按组随机抽样后执行操作。
2. 对每次评估记录迭代编号、修改类型、线长、通孔、运行时间与目标值。
3. 提供全局日志与图表输出，支持后续复盘与参数调优。
4. 当某轮后端工具执行失败或指标提取失败时，算法可跳过该轮并继续，增强流程鲁棒性。

### 4.4.3 复杂度讨论

设总轮次为 $R$，每轮迭代为 $K$，每次候选评估开销为 $\tau_{eval}$，则总时间近似：

$$
\mathcal{T}\approx R\cdot K\cdot \tau_{eval}+\mathcal{T}_{prep}
$$

其中 $\tau_{eval}$ 由工具执行主导，远大于邻域生成开销。因此，本章方法的优化重点不是降低邻域生成复杂度，而是：

1. 减少无效评估次数。
2. 提高每次评估命中高质量区域的概率。
3. 通过多轮重启提升“单位评估收益”。

### 4.4.4 参数敏感性分析

从已有实验看，以下参数对结果影响最显著：

1. 初始温度与冷却率：决定探索-收敛平衡。
2. 目标函数权重：决定线长收益与边界规整性的折中。
3. 邻域强度上限：决定跨局部盆地能力。
4. round 数量：决定额外时间投入与增益饱和点。

经验上，round 从 1 增加到 2 或 3 时通常有明显收益；继续增加时边际收益下降。

### 4.4.5 与工业工具协同的工程意义

本章流程直接利用商业工具真实反馈，避免“代理指标与真实指标不一致”问题。相较完全替换布局器，该方式优势在于：

1. 部署成本低，不破坏既有签核链路。
2. 可逐步上线，便于 A/B 对比验证。
3. 便于与不同模块化前端结果适配，具备较强方法通用性。

### 4.4.6 可复现执行规范

为保证实验可复现，本章建议采用如下执行规范：

1. 固定脚本版本与参数模板，所有实验记录统一归档。
2. 每次运行输出完整日志，包括修改类型、目标值变化、接受与拒绝决策。
3. 对同一设置至少执行多次，分别报告最优值、均值和标准差。
4. 保留失败迭代日志，不仅记录成功样本，避免幸存者偏差。

同时，建议建立统一结果目录结构，例如：

```text
results/
    flow1/
    flow2/
    flow3/
    flow4_sa/
    flow5_full/
    ablation/
    logs/
    figures/
```

该规范能显著降低后续论文写作阶段的结果追溯成本，也有利于后续答辩时进行可复现实验展示。

### 4.4.7 预算约束下的停止准则

除了温度下界，本章建议引入三类早停条件：

1. 连续 $N$ 轮无改进早停。
2. 单位时间增益低于阈值早停。
3. 达到预设 wall-clock 预算早停。

设第 $t$ 轮目标值为 $J_t$，可定义单位时间增益：

$$
G_t=\frac{J_{t-1}-J_t}{\Delta t}
$$

当 $G_t$ 连续低于阈值 $\epsilon$ 时，可触发提前停止。该策略在工业场景中尤其重要，因为项目更关注“在固定时间内拿到更好解”，而非无限逼近理论最优。

## 4.5 实验设置

### 4.5.1 测试基准与输入构造

实验采用 ITC'99 与脉动阵列（Systolic Array）两类设计：

1. ITC'99：用于验证在真实逻辑结构下的泛化能力。
2. 3x3、4x4、5x5 脉动阵列：用于验证在规整结构上的优化行为。

对于 ITC'99，由于原始案例不直接带模块划分，采用图划分方法将设计拆分为若干模块，再提取模块间连线关系输入模块化流程。

### 4.5.2 对比流程定义

结合二阶段报告，本章采用五类流程：

1. Flow1：固定 IO 条件下，评估基础模块化引导效果。
2. Flow2：允许 IO 正常布局，观察约束鲁棒性变化。
3. Flow3：弱化 IO 干扰，评估模块约束主导时的潜力。
4. Flow4：在 Flow3 基础上叠加模拟退火外层优化。
5. Flow5：补全设计因素后进行端到端评估。

该分层设计有助于定位收益来源，避免将不同因素混叠。

### 4.5.3 评价指标

本章重点关注以下指标：

1. 线长下降率：

$$
\text{WL Redc.}=\frac{W_{base}-W_{new}}{W_{base}}\times 100\%
$$

2. 通孔变化：评估布线层切换代价。
3. 拥塞相关指标：评估可布线性改善趋势。
4. 运行时间：评估优化收益与开销是否匹配。

### 4.5.4 实验公平性设置

为保证比较公平，实验遵循以下原则：

1. 保持同一工具版本与同一工艺设置。
2. 保持同一网表输入与时序约束。
3. 同类对比仅改变约束输入，不改变下游流程。
4. 对随机算法使用多次运行并统计最优与均值。

### 4.5.5 Flow1-Flow5 的功能定位细化

为避免概念重叠，五类流程在章节中可按下表统一描述：

| 流程 | 主要目的 | 关键控制变量 | 预期结论 |
|---|---|---|---|
| Flow1 | 验证基础引导有效性 | 固定 IO、不同约束类型 | 模块约束是否带来正向增益 |
| Flow2 | 评估完整 IO 条件下鲁棒性 | IO 参与、不同利用率 | 收益是否受 IO 显著扰动 |
| Flow3 | 评估模块约束主导潜力 | 弱化 IO 干扰 | 约束本体上限收益 |
| Flow4 | 验证模拟退火增量收益 | 在 Flow3 最优上继续搜索 | 累计收益与迭代特征 |
| Flow5 | 评估端到端可落地性 | 补全设计因素 | 实际流程下可复现收益 |

该定义有助于答辩时解释“为何需要多流程分层实验”，并减少审稿环节对实验可解释性的质疑。

### 4.5.6 参数搜索空间与默认设置

结合现有脚本，本章可给出默认参数与扫描范围（写作时可放入表格）：

| 参数 | 含义 | 默认值 | 建议扫描范围 |
|---|---|---|---|
| $T_0$ | 初始温度 | 1.0 | [0.5, 2.0] |
| $\eta$ | 冷却率 | 0.99 | [0.95, 0.995] |

### 4.5.7 统计报告建议

为提升结果可信度，建议在正式版第四章补充以下统计量：

1. 每组实验的均值与标准差。
2. 最优值与中位值的差异。
3. 显著性检验或置信区间。

这样可以避免“只报告最优样本”带来的偏差，使实验结论更具说服力。

## 4.6 实验结果与分析

### 4.6.1 Flow1：基础引导效果

在 Flow1 条件下，ITC'99 与脉动阵列均出现可观改进案例。统计显示，在 14 个代表案例中，11 个案例线长获得正向优化，其中 9 个案例的优化幅度达到或超过 1%。

以 ITC'99 为例，b18 与 b19 在该流程下均获得超过 1.3% 的改进，说明模块化约束在中大规模设计中具备明确价值。与此同时，少数案例出现负增益，反映单次约束映射仍存在不稳定性，这也是后续引入模拟退火外层探索的动机。

### 4.6.2 Flow2 与 Flow3：IO 影响分析

Flow2（允许 IO 正常参与）结果显示，不同利用率下收益波动明显：

1. 在部分利用率条件下，约束收益受 IO 拖拽影响显著。
2. 当利用率较高时，局部拥塞与 IO 走线竞争加剧，可能出现负优化。

Flow3（弱化 IO 干扰）结果表明，模块化约束潜力被明显释放。在 4x4 阵列测试中，70% 与 80% 利用率下出现多组约 5% 级别优化。这说明：

1. 约束输入本身具有有效性。
2. IO 与模块约束之间确实存在协同设计空间。

因此，本章后续采用模拟退火在约束层进行再优化，以增强对不同 IO 条件的适配能力。

### 4.6.3 Flow4：SA 外层优化的增量收益

Flow4 在 Flow3 最优约束基础上继续执行 SA，可获得累计增益。以 4x4 阵列为例，已有结果显示：

1. 60% 利用率累计优化约 7.03%。
2. 70% 利用率累计优化约 7.76%。
3. 80% 利用率累计优化约 6.14%。
4. 90% 利用率累计优化约 0.48%。

可见，SA 在中等利用率下收益更稳定；高利用率场景受可行空间压缩影响，边际收益下降。这与退火方法在受限空间中的行为一致。

### 4.6.4 Flow5：完整流程下的端到端表现

Flow5 在补全设计因素后，仍可在多个利用率下保持正向收益。报告结果显示：

1. 不利用面积缩减时，典型收益可达 4.40%。
2. 若允许进一步面积收缩，典型收益可达 7.16%。

这说明本章外层流程并非只在理想化场景有效，而是具备进入完整流程的潜力。

### 4.6.5 典型数值结果汇总

结合当前已整理数据，可在正文中给出简明汇总表，突出可复现结论：

| 场景 | 代表结果 | 说明 |
|---|---|---|
| Flow1 总体 | 14 个案例中 11 个正收益 | 基础约束输入有效 |
| Flow1 ITC'99 | b18/b19 均超过 1.3% | 中大规模设计可获益 |
| Flow3 4x4 阵列 | 多组约 5% 级别收益 | 约束主导时潜力明显 |
| Flow4 累计收益 | 70% 利用率约 7.76% |模拟退火带来持续增量 |
| Flow5 完整流程 | 典型约 4.40% | 工程条件下仍有收益 |
| Flow5 + 面积缩减 | 典型约 7.16% | 约束协同潜力进一步释放 |

该表可作为第四章“结果总览图景”，后续再进入各子节展开。

### 4.6.6 现象分析：方形化倾向与工具偏好

在多组实验中观察到，经过模拟退火微调后，部分模块边界呈现更明显的方形化趋势。该现象可能意味着商业工具对更规整、轴对齐程度更高的约束输入有更稳定响应。

这一观察带来两个启示：

1. 目标函数中加入规整度项是合理的。
2. 后续可研究“工具友好型几何先验”，进一步提升优化稳定性。

### 4.6.7 消融实验分析

结合已有消融结果，本章关注三类机制贡献：

1. 斥力约束：移除后模块更易聚集中心，排布顺序恶化。
2. 均匀分布约束：移除后全局分布偏离常规实现形态。
3. 多阶段策略：移除后更易早期固化，后期难以跨越局部最优。

消融结论说明，本章方法不是“单一参数调优”，而是由“邻域设计 + 多阶段调度 + 规整约束”协同驱动。

### 4.6.8 失败案例与局限性

尽管总体收益明确，但本章方法仍存在边界条件：

1. 极高利用率下可行空间过小，收益不稳定。
2. IO 占比高的设计中，约束收益可能被 IO 主导项稀释。
3. 当前目标函数未显式纳入全部签核指标，需要后续扩展。
4.模拟退火本质为随机过程，仍需多次运行与统计保证稳定性。

这些局限性不影响方法有效性结论，但提示后续研究应向“多目标联合 + 自适应策略”继续推进。

### 4.6.9 结果可信度与威胁分析

为避免过度结论，本章需要明确结果威胁来源：

1. 数据集威胁：当前样本覆盖有限，仍需更大规模工业设计验证。
2. 工具版本威胁：不同版本布局器对约束解释可能存在差异。
3. 参数耦合威胁：温度与邻域强度存在耦合，若扫描不足可能低估潜力。
4. 统计威胁：若只报告最优值，可能高估实际稳定收益。

对应的缓解策略包括：扩展基准、锁定版本、分层调参与多次统计。将这些内容写入第四章可显著提升论证严谨性。

### 4.6.10 对比遗传算法的实验建议（可直接落地）

针对用户提出的“与遗传算法对比”要求，本章可采用以下统一实验协议：

1. 统一评估预算：GA 与模拟退火采用相同工具调用次数上限。
2. 统一初始约束：均从同一模块化结果出发。
3. 统一目标函数：都优化线长 + 规整度组合目标。
4. 统一报告指标：最优值、均值、方差、单位时间收益。

预期可验证的问题是：在工具调用昂贵条件下，SA 是否在“单位评估收益”上更优；而 GA 是否在“全局探索多样性”上更优。该对比将帮助明确本章方法的适用边界。

### 4.6.11 案例级结果解读模板（建议用于正式稿）

为了让实验章节更具可读性，建议在正式稿中增加“案例级解读”段落。以下给出可直接套用的结构：

1. 案例背景：说明该案例规模、利用率与 IO 特征。
2. 约束变化：给出相较基线约束在类型与几何上的主要变化。
3. 指标变化：报告线长、通孔、运行时间的相对变化。
4. 机理解释：结合布局热区与连线拓扑解释收益来源。
5. 失败复盘：若收益不显著，说明可能原因与后续改进方向。

例如对 4x4 阵列 70% 利用率场景，可按如下逻辑组织：

1. 该场景既有足够可行空间，又存在中等拥塞压力，适合作为外层优化验证对象。
2.模拟退火在早期迭代通过类型切换与局部边界平移快速降低目标值，说明粗粒度初值附近存在可挖掘改进空间。
3. 中后期迭代主要贡献来自小幅几何修整，体现出“先粗调、后细修”的退火行为。
4. 与单阶段结果相比，多轮策略的累计收益更稳定，说明重启机制有效缓解了局部最优固化。

再如对高利用率场景，可强调：

1. 约束可行域收缩导致可接受邻域规模显著下降。
2. 候选扰动较难同时满足线长改善与可行性约束。
3. 该场景更依赖精细温控与多起点策略，而非大幅度扰动。

通过这种“场景化叙事”，第四章实验部分能够从“报表罗列”提升为“机理解释”，增强学术说服力。

### 4.6.12 面向答辩的结果组织建议

2. 第二层：稳定性证据。展示多轮统计与不同利用率下趋势，证明方法可复现。

这种证据组织方式与本章方法论一致，即强调可工程复现的增量收益，而非单次最优结果。

## 4.7 与第三章拥塞感知初始布线的协同机制

第三章聚焦初始布线的拥塞感知与并行加速，本章聚焦布局侧约束输入优化。二者可形成前后协同：

1. 本章通过改善模块空间分布，降低初始拥塞热点形成概率。
2. 第三章在更优初态上执行动态拥塞感知布线，可更快收敛并减少回退。
3. 两章共同实现“布局先减压、布线再精化”的流程闭环。

从方法论看，本章属于“上游设计空间整形”，第三章属于“下游路由求解增强”。这种分层优化路线能在不依赖单一算法突破的情况下，获得更稳健的全流程收益。

进一步地，可将两章方法抽象为统一闭环：

1. 布局外层优化输出更低拥塞风险的空间约束。
2. 初始布线在该空间约束上执行动态拥塞感知优化。
3. 若布线反馈仍出现局部拥塞，可回传约束层做二次微调。

这为后续构建“布局-布线双向迭代”提供了接口基础。即使当前论文阶段尚未实现完整闭环，章节中明确该演进方向也有助于体现研究路线的连续性。

## 4.8 本章小结

本章围绕“基于模块化布局结果的外层优化”提出了一套细粒度设计空间探索流程。核心结论如下：

1. 细粒度外层优化可以有效弥补粗粒度模块化方法在工具适配与局部精修上的不足。
2. 基于合法化边界与多操作邻域的模拟退火流程，能够在可控开销下持续挖掘布局约束增量收益。
3. 多阶段与多轮重启策略可显著降低陷入局部最优的风险。
4. 在多个测试场景中，本章方法对线长与可布线性均表现出正向促进作用。

从论文主线看，本章强化了“先布局减压，再布线求精”的问题驱动逻辑：

1. 面向布局侧，提供低侵入、可落地的外层优化能力。
2. 面向布线侧，为第三章方法提供更优初态与更稳定资源分布。
3. 面向全流程，体现质量与效率协同优化的系统性思路。

需要再次强调的是：本章贡献不在于提出 Slime 核心建模或核心解析优化算法，而在于围绕已有模块化布局结果，构建并验证了可工程落地的外层优化流程。该流程与第三章动态拥塞感知初始布线方法相互补充，共同构成本文“布局-布线协同优化”主线。
